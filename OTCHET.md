# Отчёт: «Поиск похожей местности» на PostgreSQL

> Учебный проект по дисциплине СУБД. Программа хранит картинки местности в
> базе PostgreSQL и по загруженному снимку находит самое похожее место.

---

## 1. Что делает программа (простыми словами)

1. В базе данных лежат картинки местности (лес, море, город…) и для каждой —
   её короткий «отпечаток».
2. Пользователь дважды кликает по `Poisk.command` и в **диалоговом окне**
   выбирает картинку.
3. Программа считает отпечаток выбранной картинки и **сравнивает** его со
   всеми отпечатками в базе.
4. Находит самую похожую картинку и показывает результат + открывает её.

Это маленькая модель навигации: «вот снимок — где это?».

## 3. База данных

База называется **`mestnost`**. Внутри — **одна таблица** `mesta`:

```sql
CREATE TABLE mesta (
    id         SERIAL PRIMARY KEY,   -- номер записи (база сама нумерует: 1, 2, 3...)
    nazvanie   TEXT   NOT NULL,      -- название места ("Les", "More"...)
    otpechatok BIGINT NOT NULL,      -- "отпечаток" картинки (одно большое число)
    kartinka   BYTEA  NOT NULL       -- сама картинка (байты PNG-файла)
);
```

### Что означает каждый столбец

| Столбец | Тип | Что хранит |
|---|---|---|
| `id` | `SERIAL` (целое) | Уникальный номер строки. База назначает сама. Первичный ключ. |
| `nazvanie` | `TEXT` | Название места. `NOT NULL` = обязательно заполнить. |
| `otpechatok` | `BIGINT` | Отпечаток картинки (64 бита = большое целое число). |
| `kartinka` | `BYTEA` | Байты самого PNG-файла. Картинка лежит прямо в базе. |

- **`id SERIAL PRIMARY KEY`** — первичный ключ. Автоматический счётчик
  через последовательность `mesta_id_seq`. Двух строк с одинаковым `id` быть
  не может.
- **`BYTEA`** — тип PostgreSQL для бинарных данных (байтов). Здесь хранится
  содержимое картинки целиком.

### Индекс
```
"mesta_pkey" PRIMARY KEY, btree (id)
```
Создаётся автоматически для первичного ключа `id`, ускоряет поиск по номеру.

## 4. Как работает «отпечаток» (average hash)

Это единственная «умная» часть — объяснить легко:

1. Картинку уменьшаем до **8×8 пикселей** и делаем чёрно-белой.
2. Считаем **среднюю яркость** этих 64 пикселей.
3. Каждый пиксель: ярче среднего → `1`, темнее → `0`.
4. Получаем **64 нуля и единицы** — это и есть отпечаток (одно число).

**Сравнение двух отпечатков** — считаем, в скольки битах они различаются
(расстояние Хэмминга):
- `0` различий → картинки одинаковые (похожесть 100%).
- Чем больше различий (до 64) → тем меньше похожесть.

Похожесть в процентах: `(64 − различия) / 64 × 100`.

## 5. Как программа работает с базой (главные SQL-команды)

**Запись картинки в базу** (`napolnit.py`):
```sql
INSERT INTO mesta (nazvanie, otpechatok, kartinka) VALUES (%s, %s, %s);
```

**Чтение всех картинок для сравнения** (`poisk.py`):
```sql
SELECT id, nazvanie, otpechatok, kartinka FROM mesta;
```

Значения передаются через плейсхолдер `%s` — это параметризованный запрос
(защита от SQL-инъекций). Эти `%s` работают **только в программе**; в SQLPro
нужно писать настоящие значения, а не `%s`.

## 6. Как запустить (демонстрация преподавателю)

PostgreSQL уже запущен через **Postgres.app** (порт 5432).

**Запуск в один клик:** дважды кликни по файлу **`Poisk.command`**.
- Программа сама подготовит базу (создаст таблицу и наполнит, если пусто).
- Откроется окно выбора файла → выбери картинку из папки `obraztsy/`
  (или `proverka/pohozhij_les.png`).
- Программа покажет ближайшее место и откроет его картинку.

**Подсказка для защиты:** выбери `proverka/pohozhij_les.png` — программа
найдёт «Les» с похожестью **81%** (похоже, но не точная копия). Если выбрать
`obraztsy/More.png` — найдёт «More» со 100%.

## 7. Простые SQL-команды для защиты (вводить в psql или SQLPro)

> В SQLPro при подключении укажи базу **`mestnost`**.

```sql
-- структура таблицы
\d mesta

-- сколько мест в базе
SELECT COUNT(*) FROM mesta;

-- список мест с отпечатками и размером картинки
SELECT id, nazvanie, otpechatok, length(kartinka) AS bajt
FROM mesta ORDER BY id;

-- найти место по названию
SELECT nazvanie FROM mesta WHERE nazvanie = 'Les';

-- отсортировать места по размеру картинки
SELECT nazvanie, length(kartinka) AS bajt
FROM mesta ORDER BY bajt DESC;
```

Запуск psql:
```bash
psql -d mestnost
```

## 8. Вероятные вопросы преподавателя и ответы

**В: Что такое первичный ключ?**
О: Столбец `id`, который однозначно определяет строку. База сама присваивает
номера через последовательность (`SERIAL`). Двух строк с одинаковым `id` нет.

**В: Почему выбрали PostgreSQL?**
О: Это надёжная клиент-серверная СУБД с открытым кодом. Поддерживает тип
`BYTEA` для бинарных данных (картинок), транзакции и параллельный доступ.
Отдельный сервер, к которому подключаются разные клиенты (программа, psql, SQLPro).

**В: Что такое тип BYTEA?**
О: Тип PostgreSQL для произвольных бинарных данных. Мы храним в нём
содержимое PNG-файла, то есть сама картинка лежит в базе.

**В: Что такое «отпечаток» и зачем он в отдельном столбце?**
О: Короткое представление картинки (64 бита). По нему быстро сравнивать
картинки, не разглядывая их попиксельно. Храним отдельно, чтобы при поиске
читать маленькое число, а не всю картинку.

**В: Как определяете, что две картинки похожи?**
О: Сравниваем отпечатки — считаем, в скольки из 64 битов они различаются
(расстояние Хэмминга). Ноль различий — картинки одинаковые.

**В: Что такое SERIAL?**
О: Целочисленный столбец с автоматическим счётчиком. При каждой вставке
база сама подставляет следующий номер через последовательность.

**В: Как программа защищена от SQL-инъекций?**
О: Все значения подставляются через плейсхолдер `%s` (параметризованный
запрос), а не склейкой строк. Драйвер сам экранирует данные.

**В: Что такое транзакция / COMMIT?**
О: Группа изменений, применяемая целиком. После вставки картинки вызываем
`commit()`, и запись становится постоянной (часть гарантий ACID).

**В: Что будет, если загрузить картинку, которой нет в базе?**
О: Программа всё равно найдёт *самую близкую* из имеющихся и покажет процент
похожести. Низкий процент → точного совпадения в базе нет.

**В: Можно ли ускорить поиск на большой базе?**
О: Сейчас программа читает все строки и сравнивает в Python — для учебной
базы достаточно. На больших объёмах можно добавить индекс или расширение
PostgreSQL для поиска по сходству.

## 9. Что можно улучшить (если спросят «что дальше»)

- Делать сравнение прямо в SQL-запросе, а не в Python.
- Добавить столбцы с датой добавления и координатами места.
- Использовать более точный алгоритм отпечатка (pHash вместо average hash).
- Добавить таблицу «маршрутов» и связь через внешний ключ (FOREIGN KEY).
